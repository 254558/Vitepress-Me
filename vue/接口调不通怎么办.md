# 前端接口请求报错 完整排查流程
## 一、先确认：请求到底有没有发出去
打开浏览器 DevTools → Network 面板

重点核查：
- 是否存在对应请求
- 请求 URL 是否正确
- 请求方法（GET / POST）是否匹配

### 如果完全没有该请求
问题出在**前端代码**：
- 事件/生命周期未触发执行
- 逻辑条件判断未进入请求分支
- `Promise` / `async` 函数未实际调用

---

## 二、请求已发出，但状态码异常
### 1. 4xx 系列（前端侧问题）
- 400：参数错误（字段名错误、数据类型不匹配）
- 401：未登录、Token 失效/缺失
- 403：接口无访问权限
- 404：请求接口路径书写错误

排查要点：
- 请求参数格式（JSON / form-data）
- 参数字段名与后端定义是否一致
- 请求头是否正确携带 Token

### 2. 5xx 系列（后端侧问题）
- 500：后端代码逻辑报错
- 502 / 503：后端服务宕机/不可用

优雅定位话术：
> 接口返回 500，说明请求已成功抵达后端，问题属于服务端逻辑异常。

---

## 三、状态码 200，但前端拿不到数据（高频坑点）
### 1. 前后端数据结构不一致
```js
// 后端实际返回结构
{ data: { list: [] } }

// 前端错误写法
res.list

// 正确写法
res.data.list
```

### 2. axios / fetch 响应层级理解错误
```js
axios.get(...).then(res => {
  console.log(res)        // 整个HTTP响应对象
  console.log(res.data)   // 真正的后端业务数据
})
```
误区：直接使用 `res.xxx` 获取业务数据

### 3. Promise / async 异步处理不当
```js
// 错误：直接拿到 Promise 对象，不是真实数据
const data = axios.get(...) 

// 正确：await 解构获取数据
const { data } = await axios.get(...)
```

### 4. Vue 中 this 指向丢失
```js
// 错误：普通函数导致 this 指向改变
.then(function(res) {
  this.list = res.data
})

// 正确：箭头函数保留 this 指向
.then(res => {
  this.list = res.data
})
```

### 5. 数据已获取，但页面不渲染
常见原因：
- 未赋值到响应式变量
- ref 变量缺少 `.value` 赋值
- `v-if` 条件拦截渲染
- 列表 key 错误导致视图不更新

---

## 四、跨域问题
浏览器报错关键字：
`Access-Control-Allow-Origin`

原因：
- 后端未配置 CORS 跨域允许
- 本地开发 Vite / Webpack 代理配置失效

---

## 五、代理/环境配置问题
本地开发常见问题：
- 全局 `baseURL` 配置错误
- 项目 proxy 代理未生效
- 请求误发到错误环境（dev / test / prod）

---

## 六、接口通但请求地址不匹配
真实场景：后端接口正常，前端请求无效
- 前端调用老旧废弃接口
- URL 缺失 `/api` 统一前缀
- 前后端服务端口号不一致

---

## 七、终极高效排查方案
使用独立工具直接调用接口：
- Postman
- curl 命令

判断逻辑：
- Postman 有数据、浏览器无数据 → **问题一定在前端**
- Postman 也无数据 → **问题在后端接口**

---

## 面试加分标准总结话术
### 标准排查思路
我通常分三层排查接口问题：
1. **网络层**：先确认请求是否正常发出、HTTP 状态码是否合规；
2. **接口层**：核对请求参数、返回数据结构、Token 身份校验、跨域及代理配置；
3. **前端层**：重点排查异步 Promise 处理、axios 响应层级、this 指向、Vue 响应式绑定与页面渲染逻辑。

### 补充加分话术
如果接口返回 200 但页面拿不到数据，我会优先核对后端返回结构体、axios 的 data 嵌套层级，同时检查 Vue 响应式变量赋值和视图更新规则。
